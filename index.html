Github pages博客平台搭建

静态Blog框架的选择（Hexo & Jekyll）

首先说一下我的选择——Hexo。

  • Jekyll基于Ruby实现，需要搭建Ruby环境；而Hexo是基于NodeJs实现的，在windows上安装NodeJs比较简单；
  • Hexo中的Next主题，经典中的经典；
  • 两者都支持Markdown语法（这非常重要）。

为什么要搭建Blog？

我对于这个问题的回答是分享，分享自己的知识，分享自己的思考，分享自己的想法。对于计算机的学习，做笔记是一件很重要的事情，把他们写在博客里，可以让大家看到，帮助到大家，有错误的地方网友们可以指出并提出自己的修改意见，让自己成为更好。我想这是一件很快乐且很重要的事情，且坚持做一件事不是很酷嘛！！！

山近月远觉月小，便道此山大于月；若人有眼大于天， 便道山高月更阔 。

如何搭建Github pages博客

1.注册github账户

首先你得有一个Github账号，github官网：https://github.com/ 。

登入自己的账号后，

直接点击creat repository就可以了，这时候输入自己的用户名.github.io就可以看到自己的一个简单的博客，是readme.md文件中的内容。例如，我的Yunfengsun.github.io 。

2.安装 Node.js

网址：https://nodejs.org/en/

点击16.14.2 LTS下载，这个是稳定版本的，下载完成之后点击安装。

安装完成之后打开CMD（win + R）输入“node -v”，得到“v16.14.2”；输入“npm -v”，得到“8.5.0”，安装成功。

3.安装git

网址：https://git-scm.com/

点击download，点击windows，下载完成之后点击安装。

安装完成之后后打开CMD（win + R）输入“git —version”，得到“git version 2.35.1.windows.2”，安装成功。

4.安装hexo

打开CMD（win + R）输入“npm install -g hexo-cli”，回车即开始安装，输入“hexo -v”，得到“hexo-cli: 4.3.0”等一连串版本号，安装成功。

5.搭建本地博客

创建一个文件夹，命名随意（例Blog），右键文件夹或键入文件夹右键，选择Git Bash Here。

输入“hexo init”将文件夹初始化为一个博客文件夹，再输入“npm install”安装依赖；

输入“hexo g”生成网页，初始网页为hexo自带的hello world的博客。

输入“hexo s”将生成的网页放在本地服务器上，打开浏览器输入：http://localhost:4000/ ，即可看到自己的网页。ctrl + C关闭本地服务。

6.发布一篇自己的博客

在创建的文件夹下打开source/_posts，在里面新建markdown文件，例如（first.md），后输入“hexo g”，再输入“hexo s”，打开浏览器输入“http://localhost:4000/ ”，就可以看到自己创建的内容了。

重装系统遇到的问题

Win10

1.遇到MBR或GPT分区表错误问题

首先这两个分区表是不同的，MBR只支持UFEI启动模式，而GPT可以支持UEFI启动和Legcy启动模式（注：是否支持UEFI启动模式需要启动BIOS查看）。如果在选择安装介质界面遇到这个问题，使用以下方法即可：

  1. 按住“shift + f10”进入cmd窗口；
  2. 输入“diskpart”进入磁盘管理模式；
  3. 输入“list disk”，查看电脑中的磁盘，看具体容量即可；
  4. 输入“select disk *(**表示磁盘序号)”，选中需要的磁盘；
  5. 输入“list partition”，查看磁盘中的分区；
  6. 输入“selcet partition *”，选择自己系统要安装的分区；
  7. 输入“convert mbr（如果要转换为gpt格式，则替换为gpt）”，转换分区表格式；
  8. 输入“exit”，退出diskpart模式；再次输入“exit”，退出cmd窗口。

2.遇到如下错误：“计算机意外的重新启动或遇到错误。Windows安装无法继续。若要安装Windows，请单击“确定”重新启动计算机，然后安装系统。”

首先这个问题大部分是由于在安装操作系统时候出现突然断电或者其他情况导致中断产生的，大部分的第一个解决办法如下

  1. 按住“shfit + f10”，打开cmd窗口；
  2. 输入“cd /”，移动目录到主分区下；
  3. 输入“Windows\system32\oobe\”，移动目录到oobe下；
  4. 输入“msoobe”，打开msoobe；

按照上述方法会进去到“海内存知己，天涯若比邻”界面，并且无限卡住，此时强制关机，重启出现同样的错误，正确的做法如下：

  1. 按住“shfit + f10”，打开cmd窗口；
  2. 输入“regedit”，打开注册表界面，找到注册表HKEY_LOCAL_MACHINE\SYSTEM\SETUP\STATUS\ChildCompletion下SETUP.exe，双击改变数值为3，退出。
  3. 重新启动即可。

Win7

1.遇到找不到USB驱动程序，无法安装的问题？

这个问题是由于WIN7的镜像文件只含有USB2.0的驱动程序，无USB3.0的驱动程序，故无法安装，解决方法如下：
方法一：换个USB2.0的口安装，一般笔记本电脑都存在USB2.0的口；

方法二：找到intel官网（此方法只适用于intel处理器的电脑），下载win7的USB3.0驱动程序，网站在最后，可以直接点击下载，使用方法如下：

官网链接：https://downloadcenter.intel.com/zh-cn/download/25476/Windows7-USB3-0Creator

  1. 解压压缩包；
  2. 以管理员模式打开nstaller_Creator.exe程序；
  3. USB drive path选择为你系统盘的盘符；
  4. 点击“create image”后等待半小时左右（intel官方给出的时间为15min）；
  5. 在log日志中出现“Update finished”即完成安装，退出。

关于学习这件事
Date 2022/04/10

从小学到大学，算起来读了十六年书了。

    小学    学习什么
    初中    怎么去学
    高中    更好的去学习
    大学    为什么要学习

很感谢这么多年的学习，尤其是开始学习编程之后，带给我无数的惊喜和荣誉，有时候我都觉得自己很厉害（虽然这样很不谦卑）。在计算机这个社区中，你可能在其他地方很难体会到，一个比你大十几岁的人会来虚心向你请教问题，在不知道你是学生的时候，一口一口叫着你前辈，知道你是学生之后，又满口夸你有前途。
是的，很多时候我陶醉在这一些中，我开门每个人只以自己所学所懂的多少，决定自己的位置，我时常可以几天几夜为了搞懂一些代码，吃的少睡的少，搞不懂的时候说这是谁写的傻逼玩意，在搞懂的时候也说这是谁写的傻逼玩意。
当然，我也明白这一切都是暂时的，我只是比别人见得多懂得多，先了解了这一些罢了。但是，计算机的魅力不就在于我们所学的永远跟不上技术进步的速度，无穷无尽。随着一个人年龄的增长，所涉及到的深度，永远比他涉及到的广度重要得多。
我也知道，自己现在还不够优秀，有太多的东西要去学去钻研。就像以前说的那样，我清楚自己想要什么，眼前的这些称赞、议论，不会蒙蔽我的双眼。
或许有一天，我会停下自己的脚步，回头看自己，给自己一个肯定“嘿嘿，还可以吧”。
不学习，只会成为废物。可是我想成为一个技术佬，代码成为了我生命不可缺少的一部分。
我想把这场技术盛宴分享给大家，一起领略这些C语言的美。
Stay hungry; stay foolish.
